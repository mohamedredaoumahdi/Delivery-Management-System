How the System Works

Complete Order Lifecycle

1. Customer Places Order

  • Customer browses shops and products through the mobile app
  • Adds items to cart with real-time price calculation
  • Selects delivery address and payment method
  • Places order with automatic validation (minimum order amount: $10)
  • Receives instant order confirmation

2. Vendor Receives & Processes Order

  • Vendor receives real-time notification of new order
  • Reviews order details, items, and customer information
  • Accepts or rejects order based on availability
  • Updates order status as preparation progresses:
    • PENDING ‚Üí ACCEPTED ‚Üí PREPARING ‚Üí READY_FOR_PICKUP
  • Customer receives real-time status updates

3. Delivery Assignment

  • System automatically notifies available delivery agents
  • Delivery agent views available orders in their app
  • Agent accepts order and navigates to vendor location
  • Order status updates to IN_DELIVERY
  • Real-time location tracking enabled

4. Delivery & Completion

  • Agent picks up order from vendor
  • Navigates to customer delivery address
  • Marks order as DELIVERED upon completion
  • Customer can rate and review the experience
  • Payment processed (if not cash on delivery)

5. Administrative Oversight

  • Admin monitors all orders in real-time
  • Can manually assign delivery agents if needed
  • Manages vendors, customers, and delivery personnel
  • Views comprehensive analytics and reports
  • Handles disputes, refunds, and cancellations
